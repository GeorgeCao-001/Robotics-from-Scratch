项目名称: ESP32-S3 遥控器固件 (remote_controller)
版本号:   v2.1.0
更新日期: 2026-05-14

================================================================================
一、主要更新内容
================================================================================

【新增功能】
  - 新增摇杆上电自动校准流程
    setup()阶段增加calibrateJoysticks()，上电后自动采样左右摇杆中立点
    分别记录速度轴与转向轴的中心坐标，用于替代固定2048中心值

  - 新增模式切换请求标志位 FLAG_MODE_SWITCH_REQUEST(bit1)
    模式按键按下后，遥控器在短时间内发送显式切换请求
    用于通知Arduino主控执行手动/自动模式切换

【修复的问题】
  - 修复摇杆映射固定使用2048中心点导致的中立漂移问题
    原mapJoystick()默认假设ADC中心恒为2048，
    当摇杆硬件中心偏移时会导致小车在自然回中状态下仍有残余输出

  - 修复摇杆停车死区过窄导致的中立区不稳定问题
    死区从固定百分比阈值调整为基于实际采样噪声动态估算，
    中立区扩大后可显著降低轻微抖动引发的误动作

  - 修复普通手动控制包无法区分"当前模式"与"模式切换请求"的问题
    flags字段现拆分为:
      bit0 = 当前模式状态
      bit1 = 用户触发的模式切换请求
    为主控移除"自动切回手动模式"逻辑提供协议基础

【优化项】
  - 摇杆原始值读取改为均值滤波+中值滤波组合，降低ADC毛刺影响
  - 摇杆调试输出增加中心点、变化量与映射结果，便于现场校准验证
  - 心跳包与控制包统一携带显式模式切换请求窗口内的flags信息

================================================================================
二、影响范围
================================================================================

  运行时影响:
    - 摇杆自然回中时的输出稳定性明显提升
    - 低速停车和停止保持更可靠
    - 手动/自动模式切换语义更清晰，避免普通控制包被误判为切换动作

  协议影响:
    - ControlPacket.flags定义从单bit扩展为双bit语义
    - ESP32-C3接收器可透明转发，无需改动转发逻辑
    - Arduino UNO主控需同步升级至v2.1.0以完整支持显式切换请求

================================================================================
三、兼容性说明
================================================================================

  与ESP32-C3接收器兼容: 兼容v2.0.0/v2.0.1
    - 接收器仅转发flags原始值，不依赖新增bit1语义

  与Arduino UNO主控兼容:
    - 建议配套v2.1.0使用，可完整支持"仅手动触发切换模式"的新逻辑
    - 若主控仍为v2.0.x，普通控制与转发仍可工作，但不会具备本次模式切换约束

  向后兼容: 不完全兼容v2.0.x的模式切换行为
    - 摇杆控制协议主体保持兼容
    - 模式切换语义已升级，建议遥控器与主控同步更新

================================================================================
四、已知问题
================================================================================

  1. 上电自动校准阶段必须保持摇杆自然回中，
     若校准期间人为拨动摇杆，中心点可能被错误记录

  2. 模式切换仍采用单按键触发，暂无长按/双击等防误触机制

  3. 遥控器断连后的停车仍依赖小车端心跳超时策略(1000ms)
