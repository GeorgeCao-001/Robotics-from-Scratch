/*
 * ============================================================
 * ESP32-S3 遥控器 + ESP32-C3 接收器 - 完整接线指南
 * ============================================================
 * 系统组成:
 *   1. ESP32-S3 遥控器 (手持端)
 *   2. ESP32-C3 接收器 (小车端)
 *   3. Arduino UNO 小车控制器 (已有)
 *
 * 通信链路:
 *   ESP32-S3 ──ESP-NOW(2.4GHz)──→ ESP32-C3 ──UART(9600)──→ Arduino UNO
 *   ESP32-S3 ←──ESP-NOW(状态)──  ESP32-C3 ←──UART(编码器)── Arduino UNO
 *
 * 控制方式:
 *   左摇杆Y轴 → 速度控制 (前推=前进, 后拉=后退)
 *   右摇杆X轴 → 方向控制 (左推=左转, 右推=右转)
 *   模式按键  → 手动控制/自动运行 切换
 * ============================================================
 */

 * ------------------------------------------------------------------
 * 五、ESP-NOW 通信协议
 * ------------------------------------------------------------------
 *
 *  控制数据包 (遥控器 → 小车, 6字节):
 *  ┌───────┬───────┬──────┬──────┬───────┬───────┐
 *  │ Byte0 │ Byte1 │Byte2 │Byte3 │Byte4  │Byte5  │
 *  │Header │ Type  │ Vel  │ Turn │ Flags │  CS   │
 *  │ 0xAA  │ 0x01  │int8  │int8  │uint8  │ XOR   │
 *  └───────┴───────┴──────┴──────┴───────┴───────┘
 *
 *  字段说明:
 *   - Header:  固定0xAA (包起始标识)
 *   - Type:    0x01=控制数据, 0x03=心跳
 *   - Velocity:-100~100 (左摇杆Y轴, 前进/后退)
 *   - Turn:    -100~100 (右摇杆X轴, 左转/右转)
 *   - Flags:   bit0=当前模式 (0=手动, 1=自动), bit1=模式切换请求
 *   - CS:      Byte0~Byte4的异或校验和
 *
 *  状态回传包 (小车 → 遥控器, 7字节):
 *  ┌───────┬───────┬──────────┬────────┬────────┬──────────┬───────┐
 *  │ Byte0 │ Byte1 │  Byte2   │ Byte3  │ Byte4  │  Byte5   │Byte6  │
 *  │Header │ Type  │Batt×10   │ EncL   │ EncR   │ Status   │  CS   │
 *  │ 0xBB  │ 0x01  │ uint8    │ int8   │ int8   │ uint8    │ XOR   │
 *  └───────┴───────┴──────────┴────────┴────────┴──────────┴───────┘
 *
 *  UART协议 (ESP32-C3 → Arduino UNO, 文本格式):
 *    发送: "$V<vel>,T<turn>,F<flags>*\n"
 *    示例: "$V50,T0,F0*\n" (手动模式, 速度50, 转向0)
 *    示例: "$V0,T0,F3*\n"  (用户触发切换到自动模式)
 *    示例: "$V0,T0,F2*\n"  (用户触发切换到手动模式)
 *    接收: "#E<encL>,<encR>,<pwmA>,<pwmB>,\n"
 *    示例: "#E25,23,120,115,\n"
 *
 * ------------------------------------------------------------------
 * 六、MAC地址配置步骤 (重要!)
 * ------------------------------------------------------------------
 *
 *  ESP-NOW 需要知道对方的MAC地址才能通信。配置步骤:
 *
 *  步骤1: 获取ESP32-C3的MAC地址
 *    - 将ESP32-C3连接电脑
 *    - 上传以下测试代码:
 *      #include <WiFi.h>
 *      void setup() {
 *        Serial.begin(115200);
 *        WiFi.mode(WIFI_STA);
 *        Serial.print("MAC: ");
 *        Serial.println(WiFi.macAddress());
 *      }
 *      void loop() {}
 *    - 打开串口监视器, 记录MAC地址 (如: AA:BB:CC:DD:EE:FF)
 *
 *  步骤2: 获取ESP32-S3的MAC地址
 *    - 同上方法, 上传到ESP32-S3, 记录MAC地址
 *
 *  步骤3: 修改遥控器代码中的CAR_MAC
 *    - 打开 remote_controller.ino
 *    - 修改: uint8_t CAR_MAC[] = {0xAA, 0xBB, 0xCC, 0xDD, 0xEE, 0xFF};
 *    - 替换为ESP32-C3的实际MAC地址
 *
 *  步骤4: 修改接收器代码中的REMOTE_MAC
 *    - 打开 car_receiver.ino
 *    - 修改: uint8_t REMOTE_MAC[] = {0x11, 0x22, 0x33, 0x44, 0x55, 0x66};
 *    - 替换为ESP32-S3的实际MAC地址
 *
 * ------------------------------------------------------------------
 * 七、系统测试步骤
 * ------------------------------------------------------------------
 *
 *  阶段1: 通信测试 (不上车)
 *    1. ESP32-S3 和 ESP32-C3 分别连接电脑
 *    2. 配置好MAC地址后上传代码
 *    3. 打开两个串口监视器 (115200)
 *    4. 观察ESP-NOW是否配对成功
 *    5. 摇动摇杆, 观察ESP32-C3是否收到数据
 *    6. 按模式键, 观察flags是否变化

 *  阶段2: UART联调

 *    2. Arduino UNO 连接电脑, 打开串口监视器
 *    3. 摇动摇杆, 观察Arduino是否收到控制指令
 *    4. 发送 'i' 命令查看Arduino状态

 *  阶段3: 电机控制测试
 *    1. 小车悬空 (车轮离地)
 *    2. 左摇杆前推, 观察车轮前进

 *    4. 测试断连自动停车功能
 *    5. 测试电池低压报警
 *
 * ------------------------------------------------------------------
 * 八、控制模式说明
 * ------------------------------------------------------------------
 *
 *  手动控制模式 (默认):
 *    - 左摇杆Y轴: 控制前进/后退速度
 *      前推 → 前进 (速度0~100)
 *      后拉 → 后退 (速度0~-100)
 *      居中 → 停止
 *    - 右摇杆X轴: 控制转向
 *      左推 → 左转 (转向-100~0)
 *      右推 → 右转 (转向0~100)
 *      居中 → 直行
 *    - 速度大小由摇杆偏移量直接决定
 *
 *  自动运行模式:
 *    - 按模式键切换到自动模式
 *    - 小车执行预设演示序列 (前进/后退/转向/停止)
 *    - 摇杆输入被忽略 (发送vel=0, turn=0)
 *    - 仅当再次按下模式键时才切回手动模式
 *
 *  LED状态指示:
 *    - 常亮: 手动模式, 小车在线
 *    - 慢闪(800ms): 自动运行模式
 *    - 快闪(500ms): 小车离线
 *    - 慢闪(1000ms): 电池低压
 *
 * ------------------------------------------------------------------
 * 九、通信距离测试参考
 * ------------------------------------------------------------------
 *
 *  ESP-NOW 通信距离 (2.4GHz, 视环境变化):
 *  ┌──────────────────────┬───────────────────────────┐
 *  │ 环境                 │ 预估距离                  │
 *  ├──────────────────────┼───────────────────────────┤
 *  │ 室内 (无遮挡)        │ 30~50m                    │
 *  │ 室内 (有墙壁)        │ 10~20m                    │
 *  │ 室外 (空旷)          │ 100~200m                  │
 *  │ 室外 (有树木/建筑)   │ 50~100m                   │
 *  └──────────────────────┴───────────────────────────┘
 *
 * ------------------------------------------------------------------
 * 十、故障排除
 * ------------------------------------------------------------------
 *
 *  现象: ESP-NOW配对失败
 *    → 检查MAC地址是否正确 (注意字节顺序)
 *    → 确认两端WiFi通道一致 (ESP_NOW_CHANNEL)
 *    → 确认两端WiFi模式均为WIFI_STA
 *
 *  现象: Arduino收不到控制指令
 *    → 检查UART接线: ESP32-C3 TX→Arduino D5, RX←D11
 *    → 确认波特率一致 (9600)
 *    → 检查ESP32-C3 RX引脚是否串联1kΩ保护电阻
 *
 *  现象: 摇杆控制方向相反
 *    → 交换摇杆VRx/VRy接线
 *    → 或在代码中反转映射方向
 *
 *  现象: 遥控器断连后小车不停
 *    → 检查REMOTE_TIMEOUT设置 (默认500ms)
 *    → 确认Arduino端checkRemoteTimeout()正常执行
 *
 *  现象: 自动模式不执行
 *    → 确认Arduino端demoMode为true
 *    → 检查flags的bit0是否正确传递
 *    → 用串口监视器查看Arduino接收到的指令
 *
 * ============================================================
 */
