# ============================================================
# 日志轮转策略说明
# ============================================================

## 1. 轮转策略概述

### 1.1 按日期轮转
- **触发条件**: 每日 00:00 自动触发
- **文件命名**: `{项目标识}_{日期}.log`
- **示例**: `rc_2026-05-13.log`

### 1.2 按大小轮转
- **触发条件**: 单个文件达到 10MB
- **文件命名**: `{项目标识}_{日期}_{序号}.log`
- **示例**: `rc_2026-05-13_001.log`

### 1.3 归档策略
- **保留数量**: 最多保留 7 个日志文件
- **压缩方式**: 旧日志自动压缩为 .zip
- **删除规则**: 超过保留数量的最旧日志自动删除

## 2. 分类存储结构

```
logs/
├── changelog/          # 每日更新日志
│   └── *.md
├── system/             # 系统运行日志
│   └── *.log
├── error/              # 错误日志
│   └── *.log
└── debug/              # 调试日志
    └── *.log
```

## 3. 文件命名规范

### 3.1 更新日志 (changelog)
```
{项目名称}_CHANGELOG_{日期}.md
示例: remote_controller_CHANGELOG_2026-05-12.md
```

### 3.2 系统日志 (system)
```
{项目标识}_{日期}.log
示例: rc_2026-05-13.log
```

### 3.3 错误日志 (error)
```
error_{日期}.log
示例: error_2026-05-13.log
```

### 3.4 调试日志 (debug)
```
debug_{日期}.log
示例: debug_2026-05-13.log
```

## 4. 权限设置

### 4.1 Windows 权限
- **所有者**: 当前用户
- **权限级别**: 完全控制
- **继承设置**: 子文件夹继承权限

### 4.2 安全建议
- 禁止将日志文件夹设置为公共可访问
- 定期备份重要日志文件
- 敏感信息不应记录在日志中

## 5. 轮转流程图

```
日志写入 → 文件大小检查
    ↓
超过10MB? → 是 → 创建新文件(序号递增)
    ↓否
日期变更? → 是 → 创建新文件(新日期)
    ↓否
继续写入当前文件
```

## 6. 配置参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| max_size_mb | 10 | 单个文件最大10MB |
| max_files | 7 | 最多保留7个文件 |
| daily_rotation | true | 启用每日轮转 |
| compress_old | true | 压缩旧日志 |
