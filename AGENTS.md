# AGENTS.md (全局)

## 1. 系统架构

本项目采用分布式机器人架构：

- **树莓派（Raspberry Pi）**：高层逻辑（视觉、规划、决策）
- **Arduino Uno**：底层控制（电机、传感器）

**必须严格遵守分离原则。**

---

## 2. 核心原则

### 2.1 关注点分离
- 不要将视觉、控制和硬件逻辑混合在一起
- 每个模块必须只有一个职责

### 2.2 接口优先
- 所有通信必须通过定义的接口进行
- 禁止直接跨层函数调用

### 2.3 硬件隔离
- 硬件逻辑必须保持在以下位置：
  - `arduino/`
  - 或 `raspberry_pi/hardware/`

---

## 3. 通信

### 3.1 UART 协议（Raspberry Pi ↔ Arduino）
- **传输层**：UART（串口）
- **格式**：JSON

**运动控制**：
```json
{
  "cmd": "move",
  "v": 0.5,
  "w": 0.1
}
```

**状态查询**：
```json
{
  "cmd": "status"
}
```

**状态响应**：
```json
{
  "status": "ok",
  "battery": 87,
  "error": null
}
```

### 3.2 云台控制（Raspberry Pi 内部）
云台舵机由树莓派 GPIO 直接控制。水平和垂直舵机均为绝对位置舵机，硬件层接收绝对角度：

| 字段 | 说明 |
|------|------|
| `pan_abs` | 水平绝对角度，范围 `[-135, 135]`，传给 270° 水平舵机 |
| `tilt_abs` | 垂直绝对角度，范围 `[-90, 90]`，传给 180° 垂直舵机 |
| `pan_delta` | 水平本轮角度变化量，仅供调试参考 |
| `tilt_delta` | 垂直本轮角度变化量，仅供调试参考 |

控制器计算输出同时包含增量与绝对角：

```
pan_abs        — 传给水平绝对位置舵机的角度输入
tilt_abs       — 传给垂直绝对位置舵机的角度输入
pan_delta      — 本轮水平角度变化量，仅供调试
tilt_delta     — 本轮垂直角度变化量，仅供调试
```

底盘串口仍只发送 `v` 和 `w`。`w` 基于水平舵机相对 0° 的绝对偏角映射，云台角度不通过 UART 发送给 Arduino。

### 3.3 约束
- **Arduino Uno**：资源受限（2KB SRAM），JSON 解析需轻量
- **树莓派**：不直接控制电机 GPIO

运动控制语义：
- `v`：线速度命令，允许正负（`+` 前进，`-` 后退）
- `w`：角速度命令，允许正负（方向由符号决定）

## 4. 代码规范

### Python（树莓派）
- 模块化设计
- 避免全局变量
- 暴露清晰的 API
- 视觉输出建议同时提供像素量和归一化量（如 `target_x_norm`, `x_error_norm`, `width_norm`）

### C++（Arduino）
- 控制循环中禁止动态内存分配
- 禁止阻塞式延时
- 使用状态机模式

---

## 5. 测试

- 每个模块必须能够独立测试
- 先进行无硬件测试
- 记录所有通信
- 对控制相关输入优先使用归一化数据进行测试，确保分辨率切换时行为一致

---

## 6. 日志

- 记录所有串口通信
- 包含时间戳
- 格式：`[TIMESTAMP] [DIRECTION] [MESSAGE]`

示例：
```
[2024-01-15 10:30:45] [TX] {"cmd":"move","v":0.5,"w":0.0}
[2024-01-15 10:30:45] [RX] {"status":"ok"}
```

## 7. 硬件配置

### 7.1 底盘
- **类型**：4WD 四轮驱动
- **电机驱动**：L298N
- **控制器**：Arduino Uno

### 7.2 计算单元
- **主控**：树莓派（Raspberry Pi）
- **摄像头**：USB 摄像头

### 7.3 云台
- **舵机**：2 个（水平 + 垂直）
- **控制**：树莓派 GPIO

## 9. 开发流程

1. **接口定义**：先定义通信协议（JSON 格式）
2. **硬件组装**：搭建底盘，连接电机驱动和 Arduino
3. **Arduino 基础测试**：单独测试电机控制（前进、后退、转向）
4. **树莓派环境配置**：安装 Python、OpenCV、MediaPipe
5. **通信联调**：验证树莓派与 Arduino 串口通信
6. **视觉模块**：实现人脸检测，输出目标坐标
7. **云台模块**：组装云台，实现人脸跟踪
8. **PID 控制**：实现跟随逻辑，调参优化
9. **集成测试**：全系统联调
10. **性能优化**：提升帧率，减少延迟

---
